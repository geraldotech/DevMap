Diferenças entre:

element > element { code... } element element{ code... }

Para contextualizar vamos usar uma imagem em 3 diferentes cenário, a ordem de chamada pode alterar os resultados, deixar o mais específico por último.

Img Global parent is body

Img parent is span, where span is parent from div

Imgs child of div / div is parent

Exemplos...

main > section works? yes, It's parent is main

<main> <p>one</p> <section> <p>two</p> </section> </main>

main > section works? no

<main> <p>one</p> <div> <section> <p>two</p> </section> </div> </main>

Tudo é questão de especificar os elementos corretamente, quando entender os parents/child do CSS tudo fica mais fácil, o img não está dentro de nenhuma tag, isso significa que seu parent é o body, e por exemplo para enconder a primeira imagem:

body > img{ display: none; }

Sem o selector `>` estariamos afetando todas as imagens, faça os testes.
